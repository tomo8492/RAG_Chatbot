# 社内文書アシスタント (RAG Chatbot)

フォルダ内の文書(PDF / Word / Excel / PowerPoint / テキスト)をもとに回答する、
**完全ローカル** の RAG チャットボットです。Ollama によるローカル LLM を使うため、
**社外へのデータ送信はありません**。

ブラウザUI(FastAPI + 自作フロント)なので、**同じネットワークの他のPCからも利用できます**。

> 旧版(Tkinterデスクトップ版)は `legacy/rag_chat_tkinter.py` に保存してあります。

---

## ✨ 主な機能

| 機能 | 説明 |
|---|---|
| 🗂 Chat / Code タブ | 画面上部のタブで2つのモードを切替。**Chat**=資料チャット、**Code**=コーディングエージェント |
| 🤖 Code(コーディングエージェント) | Claude Code 風。**作業フォルダ**内でAIがファイルの一覧/読取/作成・編集・コマンド実行を行い依頼を達成。**変更・実行は毎回プレビュー(差分)を確認して承認**したときだけ実施。モデルは画面から変更可 |
| 📁 参照資料フォルダの選択 | サーバ(このアプリを動かすPC)のフォルダをブラウザから選んでインデックス化。複数フォルダ対応 |
| 🔒 資料内のみで回答 | 参照フォルダを選択した会話では、**そのフォルダ(と添付)の資料だけ**を根拠に回答。資料に無いことは一般知識で補わず「記載が見つかりませんでした」と返す |
| 🌐 他PCからアクセス | `0.0.0.0` で待受。社内LANの他PCのブラウザから利用可能 |
| 💬 複数会話・履歴保存 | Claude風に会話を複数保持。サイドバーから切替・リネーム・削除 |
| ⏩ ストリーミング・停止・再生成 | 回答を逐次表示。途中停止、もう一度生成し直し |
| 🧠 工数(思考の深さ) | 推論モデル(qwen3等)の thinking を「最小〜最大」で調整。思考過程も表示 |
| 📝 Markdown / コード表示 | 表・箇条書き・コードを整形表示。コードと回答はコピー可能 |
| 📎 ファイル添付 | チャットに直接ファイルを添付し、その内容について質問 |
| 🖼 画像コピペ・D&D | スクショを **Ctrl+V で貼り付け** / 画像・ファイルを**ドラッグ&ドロップ**。画像はVisionモデルで内容を理解 |
| 🔎 出典表示 | 回答の根拠にした資料ファイル・場所を明示 |
| 📄 ファイル出力 | 回答を **Word / Excel / PowerPoint / HTML / テキスト / Markdown** で保存。コードは `.bas` 等で個別ダウンロード |
| ⚙ 細かな設定 | モデル / 温度 / 回答長 / top_k / コンテキスト長 / チャンク等を調整 |
| ⚡ チャット欄クイック設定 | チャット画面から モデル・工数・回答長・参照件数 を即変更 |
| 🔐 簡易パスワード認証 | LAN公開時の保護。`.env` でON/OFF |
| 🌓 ライト / ダークテーマ | 右上のボタンで切替 |

## 🛠 必要なもの

1. **Python 3.10 以上**(推奨 3.11 / 3.12)
2. **Ollama**(ローカルLLM実行環境) … https://ollama.com/download
3. 文書を入れたフォルダ

## 📦 セットアップ

### 1. Ollama の準備

```bash
# Ollama をインストール後、使うモデルを取得(例)
ollama pull gemma3:27b      # 既定。マルチモーダル(文章+画像)で1つで完結
# 「工数(思考)」を使いたい場合は推論モデルも: ollama pull qwen3:32b
```

Ollama は通常 `http://localhost:11434` で自動起動します。

### 2. このアプリの準備

```bash
# 依存パッケージのインストール(初回は torch 等のDLで時間がかかります)
pip install -r requirements.txt
```

### 3. 設定ファイル(任意)

```bash
cp .env.example .env
# .env を編集(モデル名・パスワード・ポート等)
```

`.env` を作らなくても既定値で動作します。主な設定:

| 変数 | 既定 | 説明 |
|---|---|---|
| `HOST` | `0.0.0.0` | `0.0.0.0`=他PCから可 / `127.0.0.1`=自分だけ |
| `LAN_ONLY` | `true` | ローカルLAN/このPC以外(インターネット側のグローバルIP)からのアクセスを拒否 |
| `ALLOWED_CIDRS` | (空) | `LAN_ONLY` 有効時に追加で許可するネットワーク(CIDR/カンマ区切り)。**社内が 172.36.x.x 等の非標準レンジなら必須**(例 `172.36.0.0/16`) |
| `PORT` | `8800` | ポート番号(他アプリと衝突する場合は変更) |
| `CHAT_PASSWORD` | (空) | 設定するとログイン必須。**他PC公開時は必ず設定** |
| `CHAT_MODEL` | `gemma3:27b` | 既定の生成モデル |
| `VISION_MODEL` | `gemma3:27b` | 画像付き質問で使うモデル(Gemma3はマルチモーダルでチャットと兼用可) |
| `OLLAMA_HOST` | `http://localhost:11434` | Ollama の場所 |
| `EMBED_MODEL` | `intfloat/multilingual-e5-small` | 埋め込みモデル(日本語対応) |

## ▶ 起動

```bash
python run.py
```

### Windows の場合(かんたん)

- **`start.bat` をダブルクリック**すると、必要なパッケージのインストールと起動をまとめて行います。
- または、フォルダで「ターミナルを開く」→ `python run.py`(`python` が無ければ `py run.py`)。
- VSCode / Cursor を使う場合は、`F5`(実行)で常に `run.py` が起動するよう設定済みです。

> ⚠️ IDEの「実行」ボタンは、**開いているタブのファイル**を実行します。出力パネルや
> `run.py` 以外のタブを開いたまま実行すると `FileNotFoundError` になります。
> その場合は `start.bat` か、ターミナルで `python run.py` を使ってください。

起動するとターミナルにアクセスURLが表示されます:

```

========================================================

 他のPCから  : http://192.168.x.x:8800
 認証        : 有効
 Ollama      : http://localhost:11434 / モデル gemma3:27b

```

ブラウザで表示されたURLを開いてください。

## 🤖 コーディングエージェント(ターミナル / Claude Code 風)

指定した作業フォルダの中で、ローカルLLMが **ファイル作成・編集・コマンド実行**を行い、
プログラム作成を手伝います(この機能はアプリを動かす実機の上で使います)。

```bash
python code_agent.py --folder "C:\Users\220557\Documents\myproject"
python code_agent.py --folder . --model gemma3:27b
python code_agent.py --folder ./proj --auto      # コマンドを確認なしで実行
```

- 操作はすべて **作業フォルダ内に限定**(外には書き込めません)
- コマンド実行は既定で **実行前に確認**(`--auto` で自動実行)
- 依頼を入力 → エージェントが作業。`/reset` 履歴クリア / `/exit` 終了
- ツール対応モデル(qwen3 等)が必要。精度は Claude 本体ほどではありません

## 🌐 他のPCのブラウザから使う

1. アプリを動かすPCで `HOST=0.0.0.0`(既定)にする
2. 起動時に表示される `http://192.168.x.x:8800` を、他PCのブラウザで開く
3. つながらない場合は、**サーバPCのファイアウォール**でポート(既定8800)の受信を許可:
   - **Windows**: 「Windows Defender ファイアウォール」→ 受信の規則 → 新規 → ポート → TCP 8800 を許可
   - **Mac**: システム設定 → ネットワーク → ファイアウォール
4. **公開する場合は必ず `CHAT_PASSWORD` を設定**してください(同じLANの誰でもアクセスできてしまうため)
5. **他PCで「アクセスできません(LAN制限)」と出る場合**: 社内ネットワークが標準のプライベートIP
   (10.x / 172.16〜31.x / 192.168.x)以外だと弾かれます。`.env` に
   `ALLOWED_CIDRS=172.36.0.0/16` のように**自社のレンジを追加**してください(カンマ区切りで複数可。社外は引き続き遮断)

> 参照資料フォルダは「サーバPC上」のフォルダを選びます。各PCのローカルフォルダを使いたい場合は、
> チャットの📎添付機能でファイルを送ってください。

## 📖 使い方

1. **参照資料を登録**(任意): 左下「📁 参照資料」→「フォルダを選んで追加」→ サーバ上のフォルダを選択 → インデックス作成
2. **会話で資料を有効化**: 「参照資料」一覧で、その会話で使う資料にチェック
   - チェックした会話では**その資料(と添付)の中だけ**から回答します。資料に無い質問には「記載が見つかりませんでした」と返答(一般知識では答えません)。
   - 資料に縛られず一般的な質問もしたいときは、チェックを外すか、別の会話を使ってください。
3. **質問する**: 下部の入力欄に入力して送信(Enterで送信 / Shift+Enterで改行)
4. **クイック設定**: 入力欄上部で「モデル / 工数 / 長さ / 参照件数」をその場で変更
5. **ファイル添付**: 📎ボタン、またはドラッグ&ドロップでファイルを添付して質問
6. **画像のコピペ**: スクショを **Ctrl+V** で貼り付け(または画像をD&D)→ 質問。画像理解には画像対応モデルが必要(既定の `gemma3:27b` は対応)
6. **詳細設定**: 左下「⚙ 設定」で温度・回答長・チャンク等を細かく調整(新しい会話の既定値になります)
7. **回答を保存**: 各回答の下「⬇ 保存」から Word / Excel / PowerPoint / HTML / テキスト / Markdown で出力。
   コードブロックは右上の「⬇」で `.bas` 等の拡張子でダウンロード(「〜をExcelで作って」→ 回答をExcel保存、も可能)

入力対応形式: `.pdf .docx .xlsx .pptx .txt .md .csv .tsv .log .json`
出力対応形式: `.docx .xlsx .pptx .html .txt .md` + コード(`.bas .py .js .html .sql` など任意)

### 🤖 Code(コーディングエージェント)

画面上部の **「Code」タブ**に切り替えると、Claude Code 風のコーディングエージェントを使えます。

1. **作業フォルダを選択**: 上部バーの「フォルダを選択」で、AIが作業するサーバ上のフォルダを指定(操作はこのフォルダ内に限定)。
2. **依頼を入力**: 例「README に使い方の章を追記して」「テストを書いて実行して」。
   AIが調査(`list_files / read_file / glob / grep`)→ 変更(`write_file / edit_file / run_command`)で作業します。
3. **計画モード(既定ON・Claude Code 風)**: AIがまず調査して**実行計画**を提示します。
   - **承認**すると実行フェーズへ進みます。**ファイル編集は自動適用**、**コマンド実行など重要操作は都度確認**します(各編集の差分はログに表示)。
   - 計画モードを**オフ**にすると、従来どおり「変更を許可」をオンにして、**変更を毎回承認**する直接実行になります。
4. **モデル変更**: 入力欄上部のモデル選択で使用モデルを切替できます(ツール対応モデル推奨。`qwen3` 等)。
5. **TODO進捗 / CLAUDE.md**: 多段タスクではAIが**TODOチェックリスト**を表示・更新します。作業フォルダに
   `CLAUDE.md`(または `AGENTS.md`)があれば自動で読み込み、規約として従います。
   計画・手順・差分・TODOは会話に保存され、**開き直しても再表示**されます。

検索(`grep`/`glob`)・部分編集(`edit_file`)に加え、**長時間コマンドのバックグラウンド実行**
(`run_background`/`command_output`/`stop_command`。devサーバ等)に対応。会話の文脈が長くなると
**自動で要約・圧縮**して作業を継続します。

> 本家 Claude Code との主な違いは「ローカルLLM(Ollama)で動く・使うモデルを選べる」点です。精度はモデルに依存します。
> ※ ターミナル版 `python code_agent.py --folder <作業フォルダ>` も同梱しています。

## 🔧 トラブルシューティング

| 症状 | 対処 |
|---|---|
| 「Ollama に接続できません」 | `ollama serve` が動いているか、`OLLAMA_HOST` を確認 |
| 「(モデルなし)」と出る | `ollama pull gemma3:27b` 等でモデルを取得 |
| 他PCから開けない | `HOST=0.0.0.0`か確認 + サーバPCのファイアウォールでポート許可 |
| 回答が資料を参照しない | 「参照資料」でその会話の資料にチェックが入っているか確認 |
| PDFが読めない | 画像のみのスキャンPDFはテキスト抽出不可(OCR未対応) |
| 初回起動が遅い | 埋め込みモデルの初回DL。2回目以降は高速 |

ログはすべて起動したターミナルに出力されます。

## 🔒 セキュリティ

LLM・埋め込み・ベクトルDBはすべてローカルで動作し、**外部送信はありません**。
社内ネットワーク内での利用を前提に、次の安全対策を備えています。

### 社内ネットワーク限定アクセス(既定で有効)
- `LAN_ONLY=true`(既定)のとき、接続元が **社内/ローカルLAN(プライベートIP)・このPC・
  リンクローカル**のときだけ許可し、インターネット側のグローバルIPからの接続は **403 で拒否**します。
- 判定は**接続元の実IP**で行い、`X-Forwarded-For` 等のヘッダは信用しません(なりすまし対策)。
- すべての画面・API・静的ファイルに適用されます。
- 自分のPCだけで使うなら `HOST=127.0.0.1`、社内LANの他PCにも見せるなら `HOST=0.0.0.0` のまま `LAN_ONLY=true`。
- **非標準のプライベートレンジ**(例 `172.36.x.x`。172は16〜31のみがプライベート扱い)を社内で使う場合は、
  `.env` の `ALLOWED_CIDRS` に CIDR を追加します(例 `ALLOWED_CIDRS=172.36.0.0/16`、カンマ区切りで複数可)。
  接続元の実IPで判定し、社外IPは引き続き遮断します。**IPレンジはコードに埋め込まず、git管理外の `.env` で管理**します。

### パスワード認証(LAN公開時・Code利用時は必須)
- `CHAT_PASSWORD` を設定するとログインが必要になります(未設定だとネットワーク内の誰でも利用可)。
- 認証は定数時間比較・署名付きCookie(`itsdangerous`、鍵は `data/secret.key` に自動生成)。
- **Code(コーディングエージェント)はサーバ上でコマンド実行・ファイル変更が可能**なため、
  LANに公開する場合や Code を使う場合は**必ずパスワードを設定**してください。

### Code(コーディングエージェント)の安全管理
- **作業フォルダ内に限定**(パス・シンボリックリンクを解決して範囲外を拒否)。
- **OS/システムの重要フォルダ・ドライブ直下・ユーザー共有ルート・アプリのデータ領域
  (`data/`= 鍵やDB)は作業フォルダに選べません**(選択UI・設定時・実行時・ファイル単位の4段で検証)。
- ファイル作成・編集・コマンド実行は **「変更を許可」をオンにし、差分/コマンドを確認して
  毎回承認**したときだけ実行(既定オフ=読み取りのみ)。
- 資料(RAG)取り込みでも、OS/システムやアプリのデータ領域は対象外。

### 運用上の推奨
- サーバは**専用の低権限ユーザー**で起動してください(承認したコマンドはその権限で実行されます)。
- 通信は平文HTTP(社内LAN前提)。社外公開はリバースプロキシ+TLS+認証を別途用意してください。
- ログ(起動ターミナル)に、認証・LAN_ONLY の状態と注意が表示されます。
