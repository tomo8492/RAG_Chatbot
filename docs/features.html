社内文書アシスタント

フォルダ内の文書をもとに回答する、完全ローカルの RAG チャットボット

完全ローカル ・ 社外送信なし ブラウザUI(他PCから利用可) Ollama + ChromaDB

iこれは何?

指定したフォルダ内の PDF / Word / Excel / PowerPoint / テキスト を読み込み、その内容を根拠に 質問へ回答する社内向けアシスタントです。AIの頭脳(LLM)・文書検索・データ保存はすべて このアプリを動かすPCの中だけで完結し、インターネットへ情報を送りません。 画面はブラウザUIなので、同じネットワークにある他のPCからもアクセスできます。

1主な機能

Claude のような操作感で、社内文書に特化したRAG機能を備えています。

📁

参照資料フォルダの選択

サーバ上のフォルダをブラウザから選んでインデックス化。複数フォルダに対応し、会話ごとに使う資料を選べます。

🌐

他PCのブラウザから利用

0.0.0.0 で待受。起動時に表示されるアドレスを、社内LANの他PCのブラウザで開けます。

💬

複数会話・履歴保存

会話を複数保持。サイドバーから切替・リネーム・削除でき、過去の会話も残ります。

⏩

ストリーミング・停止・再生成

回答を逐次表示。途中で停止したり、同じ質問をもう一度生成し直せます。

🧠

工数(思考の深さ)

推論モデルの thinking を「最小〜最大」で調整。じっくり考えさせることができ、思考過程も表示します。

📝

Markdown / コード表示

表・箇条書き・コードを整形表示。コードブロックや回答をワンクリックでコピーできます。

📎

ファイル添付

チャットに直接ファイルを添付(ドラッグ&ドロップ可)し、その内容について質問できます。

🔎

出典表示

回答の根拠にした資料のファイル名・場所(ページやシート)を明示します。

📄

ファイル出力

回答を Word / Excel / PowerPoint / HTML / テキスト / Markdown で保存。コードは .bas 等の拡張子で個別ダウンロードできます。

⚙

細かな設定

モデル・温度・回答長・参照件数・コンテキスト長・チャンク分割などを詳細に調整できます。

⚡

チャット欄クイック設定

入力欄の上で「モデル・工数・長さ・参照件数」をその場で切り替えられます。

🔐

簡易パスワード認証

LANに公開する際の保護。.env でパスワードを設定するとログイン画面が出ます。

🌓

ライト / ダークテーマ

右上のボタンで配色を切替。好みや時間帯に合わせて見やすく。

3セットアップと起動

必要なもの: Python 3.10+ ・ Ollama(ローカルLLM)・ 文書フォルダ

1

モデルを取得

Ollama をインストールし、使うモデルを取得します(thinking対応モデル推奨)。

2

依存をインストール

必要なパッケージを入れます。初回は埋め込みモデル等のDLで時間がかかります。

3

起動

起動するとアクセスURLが表示されます。ブラウザで開けば利用開始です。

# ① モデル取得(例)
ollama pull qwen3:8b

# ② 依存インストール
pip install -r requirements.txt

# ③ 起動
python run.py
起動後の表示例:
このPC : http://localhost:8000
他のPCから : http://192.168.x.x:8000

4他のPCのブラウザから使う

同じネットワーク(社内LAN・同じWi-Fi)にあるPCから利用できます。

手順内容
1アプリを動かすPCで HOST=0.0.0.0(既定)にする
2起動時に表示される http://192.168.x.x:8000 を、他PCのブラウザで開く
3つながらない場合は、サーバPCのファイアウォールでポート(既定8000)の受信を許可
4公開時は .env の CHAT_PASSWORD を必ず設定(同じLANの誰でも開けてしまうため)
参照資料はサーバPC上のフォルダを選びます。各PC手元のファイルを使いたい場合は、 チャットの📎添付機能でファイルを送ってください。

5使い方の流れ

操作説明
資料を登録左下「📁 参照資料」→「フォルダを選んで追加」→ サーバ上のフォルダを選択 → インデックス作成(状態が「準備完了」になればOK)
会話で資料を有効化「参照資料」一覧で、その会話で使う資料に☑チェック。ヘッダーに「📁 資料」バッジが出ます
質問する下部の入力欄に入力して送信。Enterで送信、Shift+Enterで改行
途中で止める生成中は送信ボタンが「■停止」に変わります。クリックで中断(途中までの回答は残ります)
作り直す回答の下「↻ 再生成」で、同じ質問にもう一度回答させます
ファイルを添付📎ボタン、または入力欄へドラッグ&ドロップ。添付内容について質問できます
設定を変える入力欄上のクイック設定で即変更。詳細は左下「⚙ 設定」で調整(新しい会話の既定になります)
回答を保存各回答の下「⬇ 保存」から Word / Excel / PowerPoint / HTML / テキスト / Markdown を選んでダウンロード。コードブロックは「⬇」で .bas 等の拡張子で保存。「〜をExcelで作って」と頼めば、その回答をそのままExcel化できます
対応ファイル形式: .pdf .docx .xlsx .pptx .txt .md .csv .tsv .log .json
※ 画像だけのスキャンPDFはテキスト抽出できません(OCR未対応)。

6設定項目の意味

クイック = チャット欄から即変更(その会話のみ) ／ 設定 = ⚙設定パネル(新しい会話の既定値)

項目場所説明
モデルクイック設定 回答生成に使う Ollama モデル。インストール済みモデルから選びます
工数(effort)クイック設定 思考の深さ。最小=速い/直球、最大=じっくり熟考。thinking対応モデル(qwen3等)で効果。非対応モデルは自動で通常生成に切替
回答の長さ(num_predict)クイック設定 回答の最大トークン数。短い〜最大で調整
参照件数(top_k)クイック設定 RAGで参照する資料チャンクの数。多いほど広く、少ないほど焦点的。0で資料参照オフ
温度(temperature)設定 創造性・ばらつき。低い=固く正確、高い=柔軟で多様
top_p設定 単語選択の多様性(核サンプリング)。通常は0.9前後
コンテキスト長(num_ctx)設定 モデルが一度に扱える文脈の長さ。0=モデル既定
システムプロンプト設定 AIの役割・口調・ルール。会話ごとに上書きも可能
チャンクサイズ / オーバーラップ設定(詳細) 資料を分割する単位と重なり。次回のインデックス作成・再構築から反映

7裏側の仕組み(RAG)

RAG = Retrieval-Augmented Generation(検索で補強した生成)。次の流れで動きます。

A

取り込み

フォルダ内の文書を読み込み、適度な長さに分割してベクトル化し ChromaDB に保存します。

B

検索

質問もベクトル化し、意味の近い資料チャンクを top_k 件取り出します。

C

生成

取り出した資料を根拠として LLM に渡し、出典付きで回答を生成します。

埋め込み(ベクトル化)には多言語対応の intfloat/multilingual-e5-base を既定で使用し、 日本語の文書・質問にも対応します。すべてローカルで処理されます。

8困ったときは

症状対処
「Ollama に接続できません」ollama serve が動いているか、OLLAMA_HOST を確認
「(モデルなし)」と表示ollama pull qwen3:8b などでモデルを取得
他PCから開けないHOST=0.0.0.0 か確認 + サーバPCのファイアウォールでポートを許可
回答が資料を参照しない「参照資料」でその会話の資料にチェックが入っているか確認
PDFが読めない画像のみのスキャンPDFは抽出不可(OCR未対応)
初回起動が遅い埋め込みモデルの初回ダウンロード。2回目以降は高速
処理ログ・エラーはすべて、起動したターミナルに出力されます。

9セキュリティとプライバシー

🔒

外部送信なし

LLM・埋め込み・ベクトルDBはすべてローカル。文書や質問が社外へ出ることはありません。

🔑

公開時はパスワードを

他PCに公開する場合は CHAT_PASSWORD を必ず設定してください。

🛡

信頼できる範囲で

フォルダ閲覧はサーバのファイルシステムを参照します。信頼できるネットワークで利用してください。
