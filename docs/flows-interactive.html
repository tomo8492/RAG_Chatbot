処理フロー(対話型ノードグラフ)

社内文書アシスタント — チャット(RAG) と Code エージェント の処理を実コードに基づき図示。

点線矢印=例外/異常系。図が描画されない環境では各枠内のテキスト定義をそのままお読みください。

💬 チャット: 質問応答フロー

質問 → 検索(密+語彙) → 出典確定 → 生成 → 表示/保存。参照フォルダ選択時は資料内のみで回答(strict-RAG)。

flowchart TD
  c_in(["質問入力
(添付・画像可)"]) --> c_rw["クエリ書き換え
(追問→自立した検索文)"]
  c_rw --> c_ret["ベクトル検索
(Chroma・候補は top_k×4)"]
  c_ret --> c_fuse["BM25 + RRF 融合
重複除去・ソース多様化・足切り"]
  c_fuse --> c_rr{"リランク有効?"}
  c_rr -->|"ON"| c_llmrr["LLMリランク
(関連度を採点して並べ替え)"]
  c_rr -->|"OFF"| c_zero
  c_llmrr --> c_zero{"ヒット 0件?"}
  c_zero -->|"はい(書き換え時)"| c_retry["原文クエリで再検索(保険)"] --> c_src
  c_zero -->|"いいえ"| c_src["出典確定
原文チャンク + 🖼 紐づく図"]
  c_src --> c_msg["プロンプト構築
strict-RAG / num_ctx 予算管理"]
  c_msg --> c_vis{"画像添付あり?"}
  c_vis -->|"はい"| c_vmodel["Vision モデルへ切替"] --> c_gen
  c_vis -->|"いいえ"| c_gen["Ollama 生成
(ストリーミング・思考分離)"]
  c_gen --> c_post["後処理 + DB保存
(出典・図も保存)"]
  c_post --> c_ui["表示: 本文 + 📎出典 + 🖼図サムネ"]
  c_ui --> c_exp["⬇ 保存(任意)
参考図を埋め込み出力"]
  c_gen -.->|"Ollama不通/モデル無し"| c_err["エラー表示(対処つき)"]

📁 チャット: 資料取り込み(チャンク化)フロー

フォルダ選択(共有サーバ可)→ 抽出 → OCR/図 → 構造分割 → 埋め込み → 保存。再構築は増分(変更ファイルだけ)。

flowchart TD
  i_pick(["フォルダ選択
ローカル / 共有サーバ(UNC)"]) --> i_guard{"全フォルダに到達できる?"}
  i_guard -->|"いいえ(共有切断等)"| i_abort["構築を中断
既存チャンクは保持"]
  i_guard -->|"はい"| i_scan["対応ファイルを再帰走査"]
  i_scan --> i_diff{"変更あり?(増分判定)
更新時刻+サイズ署名"}
  i_diff -->|"変更なし"| i_keep["既存チャンク据え置き"]
  i_diff -->|"新規/変更"| i_load["本文抽出
PDF頁 / Word節 / Excelシート / PPT"]
  i_load --> i_ocr{"テキスト層が無い頁?"}
  i_ocr -->|"はい(スキャン頁)"| i_vlm["OCR(VLM/tesseract)
画像非対応モデルは即検知"]
  i_ocr -->|"いいえ"| i_img
  i_vlm --> i_img["図の抽出・保存
data/doc_images/ へ"]
  i_img --> i_fig["図チャンク生成
VLMが説明文を索引化"]
  i_fig --> i_split["構造分割
見出しパス + chunk/overlap"]
  i_split --> i_ctx{"文脈付き埋め込み ON?"}
  i_ctx -->|"ON"| i_doc["文書の主題1〜2文を生成し
埋め込みに前置き"] --> i_embed
  i_ctx -->|"OFF"| i_embed["埋め込み(e5・128件バッチ)"]
  i_embed --> i_store["Chroma 保存
削除ファイルのチャンク除去"]
  i_keep --> i_store

🤖 Code エージェント フロー

調査 → 計画承認 → 実行(編集は承認/自動適用) → 自律検証。すべて作業フォルダ内に限定。

flowchart TD
  a_in(["依頼入力
作業フォルダ + モデル/設定"]) --> a_ws{"作業フォルダは安全?
(システム領域・.env 等を拒否)"}
  a_ws -->|"NG"| a_deny["拒否(理由を表示)"]
  a_ws -->|"OK"| a_ctx["文脈の構築/復元
CLAUDE.md 読込 + 自動圧縮"]
  a_ctx --> a_gen["LLM生成(ストリーム)
ツール呼び出し / 本文JSON復元"]
  a_gen -.->|"コンテキスト超過(400)"| a_ovf["強制切り詰め→1回再試行
ダメなら num_ctx 増の案内"]
  a_ovf -.-> a_gen
  a_gen --> a_branch{"ツール呼び出し?"}
  a_branch -->|"なし"| a_verify{"変更済み&未検証&検証ON?"}
  a_branch -->|"present_plan"| a_plan["計画カード→ユーザー承認"]
  a_plan -->|"承認"| a_exec["実行フェーズへ
(以後の編集は自動適用)"] --> a_gen
  a_plan -->|"却下"| a_done
  a_branch -->|"ask_user"| a_ask["選択式の質問カード
(調査前の乱用はガード)"] --> a_gen
  a_branch -->|"explore"| a_sub["調査サブエージェント
読み取り専用・別文脈"] --> a_gen
  a_branch -->|"読み取り系"| a_read["list / read / glob / grep /
summarize_path"] --> a_gen
  a_branch -->|"変更系"| a_gate{"対象を read 済み?"}
  a_gate -->|"未読"| a_back["差し戻し
まず read_file を要求"] --> a_gen
  a_gate -->|"読了"| a_conf{"承認は必要?
(モード・自動適用設定による)"}
  a_conf -->|"確認カード"| a_user{"ユーザー判断"}
  a_user -->|"承認 / 以後自動"| a_apply
  a_user -->|"理由つき拒否"| a_rej["拒否理由をモデルへ伝達"] --> a_gen
  a_conf -->|"自動適用"| a_apply["適用 + undo控え
+ 構文チェック"]
  a_apply -->|"結果をモデルへ(構文エラーは差し戻し)"| a_gen
  a_verify -->|"はい"| a_run["検証コマンド実行
(設定 or 自動検出・最大3回)"]
  a_run -->|"失敗→修正させる"| a_gen
  a_run -->|"成功"| a_done["完了(done)
本文+ステップをDB保存"]
  a_verify -->|"いいえ"| a_done
